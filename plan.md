Improve LaTeX Preview to Flawless PDF Render
Goal Description
The current frontend live preview uses an in-browser helper (
src/lib/latex-to-html.ts
) that attempts to manually parse LaTeX to HTML using RegEx and KaTeX. This approach fundamentally fails for complex LaTeX documents, resulting in a "super cheap" look.

To fix this, we will remove the frontend HTML parsing entirely, and instead wire the preview to use the backend latex-engine (which already generates perfect PDF output) either on-demand (by clicking a Re-compile button) or automatically on a debounce (simulating Overleaf's live-preview behavior).

User Review Required
No user review is required for this plan. This plan is designed to be highly specific and actionable for an AI agent to execute.

Detailed Step-by-Step Instructions:

Import useEffect from react and useSubmitConversion from @/lib/hooks.

Add a useEffect that listens to sourceContent. When it changes, start a timeout (e.g., 2500 milliseconds).
If the timeout triggers, and the content is valid (not empty), dispatch the compilation:
typescript
useEffect(() => {
  if (!sourceContent) return;
  
  const timeoutId = setTimeout(() => {
    const file = new File([sourceContent], "document.tex", { type: "text/plain" });
    submitMutation.mutate({
      file,
      options: { direction: "latex-to-pdf" },
    });
  }, 2500);
  return () => clearTimeout(timeoutId);
}, [sourceContent]);
// Note: Be careful not to include `submitMutation.mutate` in the dependency array directly if it causes infinite loops, or use an immutable ref to trigger it.
(Edge Case) Ensure the debounce doesn't constantly trigger if the content hasn't semantically changed or if it's the initial load. You might want to track the last compiled content string in a useRef to only trigger the mutation if sourceContent !== lastCompiledContent.current.
